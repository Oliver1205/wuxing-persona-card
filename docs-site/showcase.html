01

用户愿意开始

首页用“出生年月 + 5 道题 + 结果可分享”降低决策成本，首屏就能看到开始测试和样例人格卡。

02

答题不再像填表

测试页从长表单升级为逐题卡片流，年份支持快速选、精确调和手动输入，移动端操作更自然。

03

结果有分享理由

结果页前置身份名、关键词、主副五行比例和“最像你的 3 个表现”，让用户更容易觉得“像我”。

04

传播能被观察

每个结果绑定短码，朋友访问 `/s/{shortCode}` 后回到结果页，后台能看到漏斗、来源和短链访问。

Backend Story

面试时讲“短链热路径”，不是只讲用了 Redis

短链传播是峰值入口。当前设计让 `/s/{shortCode}` 优先走 Redis 短码映射， 命中时直接拿 resultId 做 302；访问事件只入有界队列，后台 worker 批量写库； 后台 overview 使用短缓存，避免运营刷新拖慢用户跳转。

302 用户跳转优先 Redis 短码映射与空值缓存 Batch 异步事件批量落库 P95 性能 smoke 阈值可配置

5 分钟讲解主线

  1. 产品闭环：匿名测算、结果页、分享链接、朋友回流、后台统计。
  2. 工程亮点：ShortLinkProvider 隔离 internal / external，Redis 与 MySQL 都可降级。
  3. 抗峰值：短链跳转与统计写入解耦，后台查询不阻塞用户 302。
  4. 边界诚实：当前是单机商业化作品，不夸大成完整分布式 SRE 系统。
