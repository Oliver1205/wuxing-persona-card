# v0.9 稳定性与隐私审计

v0.9 的目标是上线前做小范围稳定性和隐私加固，不新增业务功能，不扩大 MVP 边界。本轮重点复核短链入口、后台访问明细、external 访问记录和访问日志字段，确保主链路继续稳定，同时减少不必要的敏感参数留存。

## 1. 本轮边界

本轮不做：

- 朋友匹配。
- 登录注册。
- 付费系统。
- AI 深度解读。
- 复杂 BI 大屏。
- 新的外部中间件。

## 2. 加固内容

### 2.1 后台短链详情短码校验

`/api/admin/short-links/{shortCode}/visits` 现在复用短链模块的 Base62 校验：

```text
^[0-9a-zA-Z]{6,7}$
```

非法短码会直接返回 400，不进入本地访问日志查询或 external 访问记录读取。

### 2.2 Referer 隐私收敛

访问事件仍保留 Referer 用于判断来源页面，但入库前会去掉 query 和 fragment：

```text
https://example.com/result/R1?sc=abc123&token=secret#share
-> https://example.com/result/R1

/result/R2?sc=abc123#card
-> /result/R2
```

这样可以避免短链参数、临时 token 或其他查询参数长期留在 `visit_event.referer`。

### 2.3 external 访问记录空列表稳定返回

独立短链服务的 `access-record` 响应如果 `records=null`，五行后台现在会稳定返回空列表，而不是触发空指针后回退本地日志。

这能区分两种情况：

- external 正常返回，但当前筛选范围没有记录：展示 external 空页。
- external 接口失败：回退本地 `visit_event`。

## 3. 测试覆盖

新增或增强测试：

- `VisitEventServiceTest`：验证绝对和相对 Referer 都会去掉 query / fragment。
- `MvpFlowIntegrationTest`：验证后台短链详情非法短码返回 400。
- `ExternalShortLinkStatsAdapterTest`：验证 external `records=null` 时返回空列表。

已有测试继续覆盖：

- 内置短链生成、复用、冲突重试。
- 无效短链空值缓存。
- `/s/{shortCode}` 302 跳转。
- PV / UV / UIP 统计。
- external 创建失败降级。
- external 统计和访问明细失败回退。

## 4. 风险复核结论

| 风险点 | 当前处理 |
| --- | --- |
| 明文 IP / clientId 入库 | 五行项目只保存 hash |
| Referer 查询参数泄露 | v0.9 已去 query / fragment |
| 非法短码打到后台查询 | v0.9 已统一校验 |
| 无效短码反复打库 | 内置短链已有 Redis 空值缓存 |
| external access-record 空 records | v0.9 已稳定返回空页 |
| external 服务失败 | 保持回退本地统计和访问日志 |
| 生产默认密钥占位 | v0.7 部署预检已检查 `.env` |
