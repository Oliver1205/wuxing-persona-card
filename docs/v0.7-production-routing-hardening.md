# v0.7 生产路由与部署加固

v0.7 的目标是把短链接入从“本地可联调”推进到“生产路由可落地”。本轮不改变默认 Docker Compose 的 internal 路由，避免破坏 v0.1-v0.6 主链路；新增的是上线前可复用的 Nginx 路由示例和部署预检脚本。

## 1. 新增内容

- `deploy/nginx.shortlink-routing.example.conf`：生产短链路由示例。
- `scripts/deploy-preflight.sh`：上线前环境变量预检脚本。
- `docs/deploy.md`：补充短链子域名、同域 `/s/**` rewrite 和预检流程。

## 2. 推荐生产路由

首选短链子域名：

```text
your-domain.com   -> 五行 H5 / API / admin
s.your-domain.com -> 独立短链服务
```

原因：

- 避免外部短链服务的 `GET /{shortCode}` 抢占 H5 根路径。
- Nginx 规则更简单。
- 短链域名可独立配置 CDN、HTTPS 和访问日志。
- 五行后端仍可保留 `/s/{code}` 作为 internal 模式和兼容入口。

备选同域路由：

```nginx
location /s/ {
    rewrite ^/s/(.*)$ /$1 break;
    proxy_pass http://external_shortlink;
}
```

这种方式只在无法使用短链子域名时采用。

## 3. 环境变量预检

上线前执行：

```bash
cp deploy/.env.example deploy/.env
# 编辑 deploy/.env
scripts/deploy-preflight.sh deploy/.env
```

预检会检查：

- 必填项是否存在。
- `APP_BASE_URL` 是否仍是示例域名。
- `ADMIN_TOKEN`、`HASH_SALT`、数据库密码是否仍是占位值。
- `SHORT_LINK_MODE` 是否为 `internal` 或 `external`。
- external 模式下短链服务地址、分组和短链域名是否配置完整。

## 4. external 模式建议

```text
APP_BASE_URL=https://your-domain.com
SHORT_LINK_MODE=external
SHORT_LINK_EXTERNAL_BASE_URL=http://shortlink:8003
SHORT_LINK_EXTERNAL_GROUP_ID=wuxing_persona
SHORT_LINK_EXTERNAL_DOMAIN=s.your-domain.com
SHORT_LINK_EXTERNAL_STATS_ENABLED=true
```

`SHORT_LINK_EXTERNAL_DOMAIN` 必须与独立短链服务生成的 `fullShortUrl` 域名口径一致，否则五行后台会主动跳过 external 统计和访问记录读取，回退本地数据。
