# v0.5 外部短链访问明细接入

## 1. 目标

v0.4 已让后台短链列表读取外部短链服务 PV / UV / UIP。v0.5 继续补齐短链详情页：external 模式下优先读取独立短链服务访问记录，让后台详情也能看到外部访问明细。

## 2. 接入接口

外部接口：

```text
GET /api/short-link/v1/stats/access-record
```

查询参数：

```text
fullShortUrl=nurl.ink:8003/abc123
gid=wuxing_persona
enableStatus=0
startDate=2026-06-09
endDate=2026-06-10
current=1
size=20
```

返回分页字段按外部 MyBatis-Plus Page 结构解析：`current`、`size`、`total`、`records`。

## 3. 五行后台行为

`GET /api/admin/short-links/{shortCode}/visits`：

1. 查询五行本地 `short_link` 业务绑定。
2. 当 `SHORT_LINK_MODE=external`、`SHORT_LINK_EXTERNAL_STATS_ENABLED=true` 且短链 domain 匹配时，请求外部 `access-record`。
3. 外部请求成功时返回 `statSource=external` 的访问记录。
4. 外部失败、internal 模式、domain 不匹配或 stats 未启用时，回退五行本地 `visit_event`。

## 4. 隐私映射

外部短链服务返回 `ip` 和 `user` 明文字段。五行后台不透传明文：

- `user` -> `clientIdHash = sha256(user + HASH_SALT)`
- `ip` -> `ipHash = sha256(ip + HASH_SALT)`
- `browser/os/network/device/locale` -> `userAgentHash` 的指纹来源
- `referer` 仅保留 `uvType/browser/os/network/device/locale` 这类非明文 IP 信息

## 5. 前端展示

短链访问详情表新增“来源”列：

- `local`：五行本地 `visit_event`
- `external`：独立短链服务 `access-record`
