# v2.6 卡片式问答体验优化

日期：2026-06-11

## 目标

v2.6 聚焦测试页的核心转化体验：让用户不再面对瀑布式长表单，而是像刷一组轻量卡片一样完成出生信息和 5 道价值取向题。

本阶段不新增后端业务逻辑，不修改测算规则，不做登录、付费、AI 解读或朋友匹配。

## 产品原则

- 降低一次性信息压力：一屏只处理一个任务。
- 让年份选择兼顾高频选择和精确控制。
- 不暴露五行属性提示，避免用户按结果倒推选择。
- 保留返回和步骤跳转，方便用户修正答案。

## 本次改动

### 出生年份精确控制

- 保留高频年份快捷按钮。
- 新增 `+1` / `-1` 微调按钮。
- 新增手动输入年份框。
- 年份会被限制在 1900 到当前年份之间。

### 卡片式问答流程

- 出生信息作为第 0 张卡。
- 5 道问题改为一题一张卡。
- 顶部步骤条展示“基础 + 1~5 题”。
- 底部操作区统一控制“上一张 / 下一题 / 生成我的人格卡”。
- 未完成当前题时阻止进入下一题，并给出轻提示。

### 选项顺序与文案克制

- 移除选项下方的“金系倾向 / 木系倾向”等属性提示。
- 每道题的选项顺序用稳定 hash 打散，避免固定五行顺序被用户察觉。
- 选项只保留答案本身和“选择 / 已选”反馈。
