# v0.8 后台运营可读性增强

v0.8 的目标是在不做复杂 BI 大屏的前提下，让 `/admin` 更适合上线前观察主链路效果：能看每日趋势、热门维度、最近结果和短链效果，同时保持 v0.1-v0.7 的短链创建、访问、统计和 external 回退能力不回退。

## 1. 本轮边界

本轮只增强后台可读性，不新增一期禁止功能：

- 不做朋友匹配。
- 不做登录注册。
- 不做付费。
- 不做 AI 深度解读。
- 不做复杂 BI 大屏。
- 不改变短链 Provider 选择和 external 失败回退策略。

## 2. 后端新增内容

`GET /api/admin/overview` 新增 `dailyTrends`：

```json
[
  {
    "date": "2026-06-09",
    "pv": 12,
    "resultCreated": 3,
    "shortLinkCreated": 3,
    "shortLinkVisits": 5
  }
]
```

趋势口径：

- 默认展示最近 7 天。
- 如果传入 `startDate/endDate`，按筛选范围展示。
- 为避免后台单次查询过宽，趋势最多返回 14 天。
- 单日 PV 来自 `visit_event`。
- 单日结果生成来自 `user_result`。
- 单日短链生成来自 `short_link`。
- 单日短链访问来自 `SHORT_LINK_VISIT` 事件。

## 3. 前端新增内容

`/admin` 增加四类运营观察区：

- 日趋势：展示 PV、结果生成、短链生成、短链访问。
- 热门星官：与已有热门五行组合并列展示。
- 最近结果：展示 resultId、五行组合、星官和创建时间。
- 最近短链：展示短码、resultId、PV 和统计来源。

这样后台不只是数字卡片，还能回答三个上线初期最常见的问题：

1. 最近几天访问是否有变化。
2. 用户生成了哪些结果和星官。
3. 哪些短链正在被访问，统计来自本地还是外部服务。

## 4. 工程取舍

本轮没有引入独立统计表、消息队列或前端图表库，原因是：

- MVP 数据量有限，后台查询可先走现有表聚合。
- 日趋势最多 14 天，避免无限范围带来查询压力。
- 文字表格比复杂图表更稳定，也更容易在面试中讲清楚统计口径。
- external 统计仍只用于短链列表和访问明细，本轮不改变外部服务职责。

后续如果真实访问量上来，可以把趋势统计拆成定时聚合表，而不是在 v0.8 提前做过度架构。

## 5. 测试覆盖

后端集成测试补充：

- overview 默认返回 `dailyTrends`。
- 日期筛选为当天时，`dailyTrends[0]` 与总览指标一致。
- 日期筛选为未来日期时，趋势指标全部为 0。
- 原有创建结果、短链访问、短链列表、访问明细仍然通过。

## 6. 验收命令

```bash

```

该命令会覆盖：

- `git diff --check`
- 后端测试
- 前端构建
- Compose 配置校验
- 构建产物提交检查
- 用户可见源码文案边界扫描

## 7. 后续

v0.9 建议进入稳定性与隐私审计：

- 无效短码和缓存穿透场景复核。
- external 统计与访问明细失败回退复核。
- 日志和外部短链明文访问数据风险复核。
- Docker Compose 全链路补验。
