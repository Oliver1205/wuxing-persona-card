# v1.2-v1.4 生产质量增强包

日期：2026-06-10

目标：在不扩大 MVP 业务边界的前提下，把 v1.1 的外部短链接入准备继续推进为更完整的生产质量能力，覆盖 CI/CD、运行态治理、后台运营工具、安全隐私加固、可选容器集成测试和分享图体验。

## 1. 范围

本阶段包含：

- v1.2：GitHub Actions 质量门禁、Docker smoke 脚本、external 短链运行态状态接口。
- v1.3：后台短链关键词筛选、来源筛选、CSV 导出、安全响应头、后台 token 常量时间比较。
- v1.4：Testcontainers 可选 MySQL 集成测试 profile、结果页分享图生成。

本阶段不包含：

- 朋友匹配。
- 登录注册。
- 付费。
- AI 深度解读。
- 复杂 BI 大屏。
- 服务器真实部署。

## 2. CI/CD

新增 GitHub Actions：

```text
.github/workflows/quality-gate.yml
```

包含两个 job：

| Job | 内容 |
| --- | --- |
| `quality` | Java 17、Node 20、`npm ci`、`scripts/quality-check.sh` |
| `container-integration` | `mvn -q -f backend/pom.xml -Pcontainer-it verify` |

`quality` 继续复用本地统一质量门禁，避免 GitHub Actions 和本地脚本各写一套逻辑。

## 3. Docker Smoke

新增：

```text
scripts/docker-smoke-test.sh
```

容器启动后可执行：

```bash
BASE_URL=http://127.0.0.1:8088 \
ADMIN_TOKEN=<your-admin-token> \
scripts/docker-smoke-test.sh
```

脚本会验证：

1. `GET /api/health`。
2. `GET /api/questions`。
3. `POST /api/results`。
4. `GET /s/{shortCode}` 302。
5. `GET /api/admin/overview`。

## 4. External 运行态治理

新增后台接口：

```http
GET /api/admin/external-shortlink/status?probe=false
X-Admin-Token: <admin-token>
```

返回内容：

- 当前短链模式。
- external 是否开启。
- external stats 是否开启。
- fallback 是否开启。
- external baseUrl、domain、groupId。
- 可选探测结果、HTTP 状态码和检查时间。

当 `probe=true` 时，后端会用配置的超时时间对 external baseUrl 做一次轻量 GET 探测。探测失败不会影响主链路，只作为后台运行态提示。

## 5. 后台运营工具

短链列表新增筛选：

- `keyword`：匹配短码或 resultId。
- `statSource`：`local` / `external`。
- 日期筛选沿用 v0.3。

新增导出：

```http
GET /api/admin/short-links/export
```

导出字段：

```text
shortCode,resultId,shortUrl,elementCombo,starOfficerName,pv,uv,uip,statSource,createdAt,lastVisitAt
```

说明：

- 导出最多 500 条。
- CSV 会加 UTF-8 BOM，便于表格软件打开中文。
- 对 `= + - @` 开头的单元格加前缀，降低 CSV 公式注入风险。
- `statSource` 是计算字段，来源筛选最多扫描最近 500 条短链，适合 MVP 后台运营排查。

## 6. 安全与隐私

新增后端安全响应头：

```text
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
Referrer-Policy: no-referrer
Permissions-Policy: camera=(), microphone=(), geolocation=()
```

后台 token 校验从普通字符串比较改为 `MessageDigest.isEqual` 常量时间比较，减少 token 比较时间差异带来的侧信道风险。

隐私边界保持：

- 不保存明文 clientId。
- 不保存明文 IP。
- 不保存明文 User-Agent。
- Referer 去 query 和 fragment。
- external 访问明细中的 IP / user 继续 hash 后展示。

## 7. Testcontainers

新增可选集成测试：

```text
backend/src/test/java/com/wuxing/persona/MvpMySqlContainerIT.java
```

运行方式：

```bash
mvn -q -f backend/pom.xml -Pcontainer-it verify
```

该测试使用真实 MySQL 容器和 `db/schema.sql`，验证：

- 创建测算结果。
- 生成短链。
- 短链 302。
- 后台短链关键词筛选。

默认 `mvn test` 不运行 `*IT`，避免本地没有 Docker 时影响普通开发。

## 8. 分享图

- 不引入新依赖。
- 基于当前结果数据生成主副五行、星官、关键词、亮点和短链。
- 继续保留短链复制能力。
