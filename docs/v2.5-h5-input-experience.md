# v2.5 H5 输入体验优化

日期：2026-06-11

## 目标

v2.5 聚焦测试页的第一段输入体验，把“出生信息”和“价值取向题”从普通表单升级为更适合年轻用户的移动端触控体验。

本阶段不新增业务逻辑，不改变后端接口，不引入登录、付费、AI 解读或朋友匹配。

## 设计判断

目标用户以大学生和年轻人为主，前端气质应当更轻、更顺、更像人格测试 H5，而不是后台表单。

字体采用不依赖外网的中文 UI 字体栈：

```text
HarmonyOS Sans SC -> MiSans -> PingFang SC -> Inter -> Noto Sans SC -> system fallback
```

选择这个方案的原因：

- 中文显示更清爽，适合移动端阅读。
- 不依赖 Google Fonts 或 CDN，线上部署更稳定。
- HarmonyOS Sans / MiSans / PingFang SC 更贴近年轻用户熟悉的手机系统视觉。

## 本次改动

### 出生年份

- 移除超长年份下拉框。
- 新增年份滑杆，支持从 1900 到当前年份连续滑动选择。
- 新增常用年份快捷按钮，例如 2008、2005、2002、1999、1996、1993。
- 年份区域增加大号数字反馈和刻度提示。

### 出生月份、日期、时段

- 出生月份改成横向滑动卡片，附带月份气质提示。
- 出生日期改成横向滑动卡片，保留“不透露”默认选择。
- 出生时段改成触控按钮网格，每个选项有轻提示文案。

### 价值取向题

- 答案选项从普通按钮升级为倾向卡片。
- 每个选项展示元素倾向，例如“金系倾向”“木系倾向”。
- 选中后显示“已选”，并增加更明确的高亮与层级。
- 增加 `aria-pressed`，方便后续可访问性和自动化测试。

### 测试脚本

- 更新移动端 E2E 脚本选择器，适配新的出生年份和月份控件。
- 给关键输入控件增加 `data-testid`。

## 验收

- `npm --prefix frontend run build` 通过。
- 浏览器 DOM 验收确认：
  - 测试页不再出现出生信息原生下拉框。
  - 年份滑杆、快捷年份、月份横向卡片、日期横向卡片、时段按钮均可见。
  - 选择 2002 年、8 月、傍晚后，出生信息状态变为已完成。
  - 答案选项显示元素倾向，点击后显示已选。
