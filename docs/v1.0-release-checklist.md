## 6. 质量门禁

v1.0 发布前必须通过：

```bash
scripts/quality-check.sh
```

该脚本覆盖：

- `git diff --check`
- 脚本语法检查
- 构建产物未被 Git 跟踪
- 文案边界关键词扫描
- `mvn -q -f backend/pom.xml test`
- `npm --prefix frontend run build`
- `docker compose --env-file deploy/.env.example -f deploy/docker-compose.yml config`

## 7. 已知风险

| 风险 | 处理 |
| --- | --- |
| 外部短链服务生产域名需最终确认 | v0.7 已提供子域名和同域路由方案 |
| 外部短链项目自身明文 IP 入库问题 | 已在五行侧 hash 后展示，外部项目仍需单独审计 |
| 后台 token 保护不适合复杂团队权限 | MVP 接受，后续可升级登录权限 |
| 标准 Docker Hub 构建可能受网络影响 | 已支持可配置基础镜像源，上线前再补真实环境构建 |

## 8. 发布动作

1. 合并 `feature/v1.0-stable-release` 到 `main`。
2. 确认 `scripts/quality-check.sh` 通过。
3. 推送 `main`。
4. 打标签：`v1.0.0-stable`。
5. GitHub README 展示稳定版状态。
