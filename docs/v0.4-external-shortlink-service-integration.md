# v0.4 外部短链服务级联调与统计适配

v0.4 的目标是把 v0.3 的 external HTTP 准备推进到真实服务级联调：五行后端在 `SHORT_LINK_MODE=external` 下可以调用本地独立短链项目创建短链，用户访问外部短链后跳回五行结果页，五行后台能按日期读取外部短链 PV / UV / UIP。

默认 `internal` 模式不变，v0.1 到 v0.3 的主流程继续可独立运行。

## 1. 本轮新增能力

- `RestExternalShortLinkClient` 支持调用外部短链统计接口 `GET /api/short-link/v1/stats`。
- 新增 `ExternalShortLinkStatsAdapter`，在后台短链列表中按配置读取外部 PV / UV / UIP。
- 新增 `SHORT_LINK_EXTERNAL_STATS_ENABLED` 和 `SHORT_LINK_EXTERNAL_STATS_ENABLE_STATUS`。
- 后台短链列表新增 `statSource` 字段，用于区分当前展示的是 `local` 统计还是 `external` 统计。
- `deploy/docker-compose.yml`、`deploy/.env.example` 和部署文档补齐 external stats 配置。
- 完成本地服务级联调，验证外部短链创建、302 跳转、五行本地业务绑定和外部统计读取。

## 2. 架构边界

```text
五行 H5
  -> POST /api/results
    -> 五行后端保存 user_result
    -> ShortLinkProvider
      -> internal: 五行内置短链
      -> external: 独立短链服务创建短链
    -> 五行后端保存 short_link 业务绑定

用户访问短链
  -> 外部短链服务 /{shortCode}
  -> 302 到五行结果页 /result/{resultId}
  -> 外部短链服务记录 PV / UV / UIP

五行后台 /admin
  -> 读取五行本地业务绑定
  -> external stats enabled 时读取外部短链统计
  -> 失败时回退本地统计
```

五行项目必须继续保留本地 `short_link` 业务绑定，因为外部短链服务只知道短链基础设施信息，不知道五行主副元素、星官、结果页业务归属和后台展示字段。

## 3. 配置说明

默认仍为内置模式：

```text
SHORT_LINK_MODE=internal
SHORT_LINK_EXTERNAL_STATS_ENABLED=false
```

启用外部短链创建和统计：

```text
SHORT_LINK_MODE=external
SHORT_LINK_EXTERNAL_BASE_URL=http://shortlink:8003
SHORT_LINK_EXTERNAL_GROUP_ID=wuxing_persona
SHORT_LINK_EXTERNAL_DOMAIN=nurl.ink:8003
SHORT_LINK_EXTERNAL_FALLBACK_TO_INTERNAL=true
SHORT_LINK_EXTERNAL_STATS_ENABLED=true
SHORT_LINK_EXTERNAL_STATS_ENABLE_STATUS=0
```

本地服务级联调时，`SHORT_LINK_EXTERNAL_DOMAIN` 必须与外部短链服务的 `short-link.domain.default` 保持同一口径。五行后台只会对匹配该 domain 的 `shortUrl` 请求外部统计，避免误把内置短链发给外部系统查询。

## 4. 路由策略

v0.4 验证了两种入口的关系：

- 外部短链入口：`http://127.0.0.1:8003/{shortCode}`，由独立短链服务负责解析和统计。
- 五行兼容入口：`http://127.0.0.1:18080/s/{shortCode}`，五行后端可通过本地业务绑定继续跳回结果页。

生产推荐使用独立短链子域名：

```text
your-domain.com   -> 五行 H5 和 API
s.your-domain.com -> 外部短链服务
```

若必须使用同域 `/s/{code}`，Nginx 可 rewrite 到外部短链服务：

```nginx
location /s/ {
    rewrite ^/s/(.*)$ /$1 break;
    proxy_pass http://shortlink:8003;
}
```

## 5. 统计适配策略

五行后台短链列表先计算本地统计，再尝试读取外部统计：

```text
local stats
  -> external mode?
  -> stats enabled?
  -> shortUrl domain matches external domain?
  -> call /api/short-link/v1/stats
  -> success: use external pv/uv/uip, statSource=external
  -> failure: keep local pv/uv/uip, statSource=local
```

外部短链统计接口按时间边界查询访问日志，v0.4 在请求外部 `endDate` 时向后补一天，确保五行后台选择 `2026-06-09` 这一天时，可以覆盖当天 `00:00:00` 到次日 `00:00:00` 前的访问。

当前只把外部 PV / UV / UIP 接入短链列表。短链访问明细仍读取五行本地 `visit_event`，后续可再接入外部 `/api/short-link/v1/stats/access-record`。

## 7. 风险与后续

- 外部短链项目当前仍有自己的用户体系、分库分表配置和访问日志口径，上线前需要固定初始化脚本或单独部署文档。
- 生产环境建议优先使用短链子域名，而不是长期依赖同域 `/s/**` rewrite。
- 外部项目访问日志仍需做隐私审计，五行项目自身不会保存明文 IP。
- 后续可继续接入外部访问记录接口，让五行后台短链详情也支持外部明细来源。
