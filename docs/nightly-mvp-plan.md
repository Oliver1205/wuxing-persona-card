# 8-10 小时 MVP 开发计划

本计划用于把第一版 MVP 从骨架推进到可运行版本。实际执行中，如果某阶段提前完成，优先补测试、文档和部署验证；如果核心闭环未完成，不提前做二期功能。

## 0. 总目标

交付一个可本地运行、可 Docker 部署、可演示单人测算闭环的 MVP：

```text
/ -> /test -> POST /api/results -> /result/{resultId}
  -> 复制 /s/{shortCode}
  -> GET /s/{shortCode} 302 回结果页
  -> /admin 查看统计
```

## 1. 时间切片

### 第 0-0.5 小时：工程纪律与计划

产物：

- `docs/development-standards.md`
- `docs/nightly-mvp-plan.md`
- `docs/quality-scorecard.md`
- `docs/teaching-manual.md` 初版

验收：

- 质量规范清晰。
- 自评机制可执行。
- 后续每个模块有评分口径。

### 第 0.5-2 小时：后端基础设施

产物：

- 统一返回结构
- 全局异常处理
- 枚举
- DTO/VO
- 数据库 SQL
- Mapper
- Redis 工具封装

验收：

- 后端能编译。
- `GET /api/health` 可用。

### 第 2-4 小时：结果生成闭环

产物：

- 题目配置接口
- 五行计算服务
- 星官服务
- 文案模板服务
- 创建结果接口
- 查询结果接口

验收：

- 提交合法数据可返回完整结果。
- 非法数据返回 400。

### 第 4-5.5 小时：短链接与统计

产物：

- 内置短链生成和绑定
- `/s/{shortCode}` 302 跳转
- 访问事件记录
- PV/UV/UIP 聚合
- 管理后台 API

验收：

- 每个结果有短链。
- 短链能跳转同一结果。
- 后台能看到短链访问变化。

### 第 5.5-7.5 小时：前端 H5 和后台

产物：

- 引导页
- 测试页
- 结果页
- 短链复制
- 后台总览
- 短链列表和详情

验收：

- 用户能完整完成一次测算。
- 页面有娱乐声明。
- `/admin` 需要 token。

### 第 7.5-8.5 小时：部署与联调

产物：

- Docker Compose 完整化
- Nginx 配置
- `.env.example`
- 部署文档更新

验收：

- Compose 配置可展开。
- Nginx 路由清楚。

### 第 8.5-10 小时：测试、自评、教学手册

产物：

- 验证记录
- 质量评分
- 教学手册完整版
- README 更新

验收：

- 关键命令执行结果记录在文档或最终回复。
- 已知缺口不隐藏。
- 高价值设计点讲清楚。

## 2. 额外推进规则

如果 MVP 核心闭环提前完成，优先做：

1. 单元测试补强：五行计算、短码生成、hash 工具。
2. 教学手册补强：短链缓存、防穿透、PV/UV/UIP。
3. 部署文档补强：云服务器步骤、环境变量解释。
4. 短链外部服务适配层准备：为后续接入 `nageoffer/shortlink` 留接口。

不提前做：

- 朋友匹配
- 登录注册
- AI 解读
- 付费
- 社区
