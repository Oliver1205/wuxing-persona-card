# 五行人格卡结果页 v2 内容系统说明

## 一、这次重构的目标

结果页不再把日主、星官、主副元素、点睛元素揉成一段说明，而是按一条清晰主线展开：

我是谁 -> 为什么是我 -> 日主核心 -> 主副关系 -> 点睛元素 -> 星官意象 -> 天人特质 -> 容易卡住 -> 成长建议 -> 下一步行动。

所有用户可见文案必须面向用户说话，不出现后台字段、英文枚举、比例计数字段、宿命化判断和空泛鸡汤。

## 二、字段与区块映射

| 结果页区块 | 前端字段 | 后端来源 | 只回答的问题 |
| --- | --- | --- | --- |
| 身份总览 | personaLabel / keywords / primaryElementName / secondaryElementName | PersonaArchetypeRegistry | 用户第一眼应该记住自己是谁 |
| 生成依据 | layoutExplanation | ResultTextService.buildGenerationBasis | 这张卡由哪些信息共同生成 |
| 日主核心 | dayMasterText | DayMasterCopyRegistry + WuxingCalendarTerms.dayTone | 日主是什么、从哪里来、对应什么底色 |
| 主副关系 | primarySecondaryText | ElementVoiceRegistry.primaryReaction / secondaryCorrection / relationText | 主元素如何第一反应，副元素如何修正 |
| 点睛元素 | accentText | ElementVoiceRegistry.accentText | 哪个元素让人格多一层余味 |
| 星官意象 | starOfficerText | StarOfficerCopyRegistry | 星官提供什么传统文化锚点 |
| 天 | heavenText | ElementVoiceRegistry.innerWorldText | 内心世界如何接收信息、处理情绪、形成判断 |
| 人 | humanText | ElementVoiceRegistry.outerWorldText | 外部世界如何感受到用户 |
| 容易卡住 | strengthText | ElementVoiceRegistry.stuckText | 这个结构最容易卡在哪里 |
| 成长建议 | growthAdvice | ElementVoiceRegistry.growthAdvice | 如何把人格结构变成具体行动 |
| 分享匹配 | shortCode / shortUrl | ShareLinkBox | 只负责保存、分享、匹配，不承载分析 |

## 三、新增或固化的 registry

- `DayMasterCopyRegistry`：十天干日主固定意象与现实转译。
- `ElementVoiceRegistry`：五行主反应、副元素修正、主副关系、点睛、天人、卡点、成长建议。
- `StarOfficerCopyRegistry`：二十八宿星官的所属分组、传统意象和用户端文化锚点。
- `PersonaArchetypeRegistry`：120 个稳定人格标签与五行组合分流。

## 四、五个验收样例

下面样例用同一组出生锚点演示文案结构，重点看区块职责是否清楚，而不是现实排盘结论。

### 样例 01：水主 + 土副 + 火点睛

身份名：星岩的澜

生成依据：这张人格卡不是只看出生年份，也不是只看某一个选择。出生年月日时提供传统文化里的气质锚点，五题选择提供当下更自然的反应倾向；五题结果更偏向水属性，土属性也有一点补强。两者合在一起，才形成这张关于日主、主副元素、星官和成长建议的整体画像。本结果用于传统文化和性格娱乐参考，不构成现实决策建议。

日主核心：你的出生日换算到四柱里，对应的是「癸亥日」。其中天干「癸」就是你的日主。日主可以理解为命盘里代表“我自己”的核心符号，它不是全部性格，但像底层的气质源头，会影响你最自然的感受方式、判断方式和启动行动的方式。癸属阴水，所以你的日主核心是癸水。

日主底色：癸水的传统意象是雨露和泉脉，重在感知、细腻、吸收和深思。放到现实里，它给你的底色不是简单的元素标签，而是一种反应方式：你更习惯先接收环境、情绪和细节，让局势在心里慢慢变清楚，再决定自己该怎么行动。

主副关系：水让你的第一反应先接收信息。你会先观察环境、捕捉细节、理解关系，再决定如何行动。副元素不是第二个标签，它更像你主元素旁边的调节器。水让你先观察、吸收和理解，土则把这股流动收进河岸里。没有土，你容易想很多；有了土，你会更在意计划、责任和结果。

点睛元素：火不负责主导你，但它让这张卡多了一层亮度。它像深处忽然露出的伏火，让原本由水和土构成的结构出现一点反差和余味。所以你不是单纯在暗处观察，也不是只靠承接和落地行动；你身上还会有一点在真正重视的事情上被点燃，想表达、想证明、想被看见。

星官意象：你的星官是「虚宿」。它属于北方玄武七宿第四宿，在传统意象里常和冬水、幽静、收敛、藏蓄和留白有关。放在这张人格卡里，它不负责判断命运，而是作为一个文化锚点，帮你记住「星岩的澜」里那种深思、留白、包容的气质。

天：你的内心会先靠近水的路径：信息进来以后，你通常会先接住语气、细节和环境暗流，让局势慢慢变清楚。随后土会补上一层把反应放进责任、秩序和承接里，让反应不至于停在单纯直觉里。

人：刚认识你的人，通常会先感到你身上有水的观察力、适应感和留白。熟悉之后，他们会发现你不只是表面的这一层，土会让你多出稳定、耐心和可交付感，火则在不显眼处补上一点关键时刻会亮出来的目标感。

卡点：这个结构最容易卡住的地方，不是你没有能力，而是水用得太满时会变成信息摄入过量、反复比较。当事情压力变大，你可能会在脑子里演算太久，行动被思考拖住。这不是缺陷，而是提醒你要给自己的反应模式留一个现实出口。

成长建议：把信息落到纸面；把节奏压到地面；点亮关键动作；固定复盘出口。

### 样例 02：火主 + 金副 + 水点睛

身份名：潮镜的焰

核心结构：火让你的第一反应先被目标和热度点燃，金负责提醒标准与边界。水作为点睛，不负责主导你，却让热度和判断多一层回声，避免只凭一时上头做决定。

日主说明：日主区只解释出生日换算出的天干来源、日主五行和底层反应，不重复主副元素关系。

星官意象：星官只作为文化锚点，解释所属分组、传统意象和为什么适合放在这张卡里，不判断现实命运。

卡点与建议：火过满时容易急于证明，金过紧时容易苛刻，水点睛提醒你等回声再判断。建议把热度放进排期、先定边界标准、等回声再判断。

### 样例 03：木主 + 土副 + 火点睛

身份名：星岩的枝

核心结构：木让你先寻找生长路径，土让这些路径种进现实。火作为点睛，让你在真正重视的事情上被点燃，不只是规划，也能表达、试错和被看见。

天人特质：内心会先看见方向和延展，再用土确认可承接的节奏；外部世界会先感到你的生长感，熟悉后会看见你其实也在意稳定和交付。

卡点与建议：木过满容易路径太散，土不足时落点不稳。建议先剪枝再生长、把节奏压到地面、点亮关键动作。

### 样例 04：金主 + 水副 + 木点睛

身份名：潮生的锋

核心结构：金让你先判断标准和边界，水让判断不至于冷硬，保留观察、理解和回旋。木点睛像石缝里的青芽，让清醒和观察之外还长出表达、路径和新的可能。

天人特质：内心先分辨标准，再继续观察暗流；别人会先感到你的清醒和分寸，熟悉后会发现你也能调整路径、继续生长。

卡点与建议：金过满容易标准太高，水过多会反复校准。建议保留一点弹性、多留一层回声、留一条低成本分支。

### 样例 05：土主 + 木副 + 金点睛

身份名：银枝的台

核心结构：土让你先稳住局面，木让稳定结构里继续长出方向和表达。金点睛不负责主导，却会在关键处提醒你看见标准、边界和取舍。

天人特质：你内心先确认安全边界和责任位置，再寻找新的生长路径；别人会先感到你的稳定和可交付感，合作里会发现你不只是守成，也能逐步打开新的路线。

卡点与建议：土过满容易启动慢、把责任都扛住，木不足时表达和试错会被压住。建议降低启动成本、给反应开生门、听见那声清铃。

## 五、质量检查结果

- 后台字段泄露检查：已由 `ResultTextServiceTest` 覆盖，检查 `personaTypeId`、英文五行枚举、`dominant`、`primaryPercent`、`/5` 等不得进入用户可见文案。
- 宿命化/恐吓表达检查：已由 `ResultTextServiceTest` 覆盖，禁止“你注定”“大富大贵”“灾祸”“破财”等表达。
- 120 模板完整性：已由 `PersonaArchetypeRegistryTest` 覆盖。
- 5 个指定组合：已由 `ResultTextServiceTest.buildShouldPassFiveV2SampleCombinations` 覆盖。
- 不确定排盘兜底：已由 `ResultTextServiceTest.buildShouldUseFallbackWhenDayMasterCannotBeCalculated` 覆盖。
