# v0.3 外部短链联调准备与后台日期统计

## 1. 版本目标

v0.3 延续 v0.2 的短链适配层，不破坏默认 `internal` 模式，也不把外部短链服务强行变成启动依赖。

本版目标：

- 让 `external` 模式更接近 `/Users/linyuxiang/JavaBackend/01_Projects/shortlink` 的真实创建接口。
- 补齐外部短链服务所需的系统用户 header、短链域名、超时配置和请求体字段。
- 后台总览、短链列表、短链访问日志支持 `startDate` / `endDate` 日期筛选。
- 保留五行项目本地 `short_link` 业务绑定和本地 PV/UV/UIP 统计。
- 不做朋友匹配、登录注册、付费、AI 深度解读或复杂 BI 大屏。

## 2. external 模式联调口径

外部短链项目创建接口：

```http
POST /api/short-link/v1/create
Content-Type: application/json
username: wuxing_system
userId: wuxing-system
realName: wuxing-system
```

五行后端请求体：

```json
{
  "domain": "nurl.ink:8003",
  "originUrl": "https://your-domain.com/result/R202606090001",
  "gid": "wuxing_persona",
  "createdType": 0,
  "validDateType": 0,
  "validDate": null,
  "describe": "五行人格卡结果 R202606090001"
}
```

外部服务返回：

```json
{
  "code": "0",
  "data": {
    "gid": "wuxing_persona",
    "originUrl": "https://your-domain.com/result/R202606090001",
    "fullShortUrl": "http://nurl.ink:8003/Abc123"
  }
}
```

五行后端仍会把 `fullShortUrl` 中的短码解析出来，保存本地绑定：

```text
resultId -> shortCode -> shortUrl
```

这样 `/admin` 可以继续从五行项目自己的业务表展示结果归属、五行组合和星官，不必依赖外部短链控制台。

## 3. 配置项

```text
SHORT_LINK_MODE=external
SHORT_LINK_EXTERNAL_BASE_URL=http://shortlink:8003
SHORT_LINK_EXTERNAL_GROUP_ID=wuxing_persona
SHORT_LINK_EXTERNAL_DOMAIN=nurl.ink:8003
SHORT_LINK_EXTERNAL_FALLBACK_TO_INTERNAL=true
SHORT_LINK_EXTERNAL_SYSTEM_USERNAME=wuxing_system
SHORT_LINK_EXTERNAL_SYSTEM_USER_ID=wuxing-system
SHORT_LINK_EXTERNAL_SYSTEM_REAL_NAME=wuxing-system
SHORT_LINK_EXTERNAL_CONNECT_TIMEOUT_MILLIS=2000
SHORT_LINK_EXTERNAL_READ_TIMEOUT_MILLIS=3000
```

说明：

- `BASE_URL` 是五行后端调用外部短链服务的内部地址。
- `DOMAIN` 是外部短链项目生成短链时使用的短链域名口径。
- `FALLBACK_TO_INTERNAL=true` 时，外部服务不可用不会中断测算主流程。
- 超时配置用于避免外部服务未启动或网络慢时拖垮 `POST /api/results`。

## 4. 后台日期筛选

v0.3 为三个后台接口增加日期参数：

```http
GET /api/admin/overview?startDate=2026-06-09&endDate=2026-06-09
GET /api/admin/short-links?page=1&pageSize=20&startDate=2026-06-09&endDate=2026-06-09
GET /api/admin/short-links/{shortCode}/visits?page=1&pageSize=20&startDate=2026-06-09&endDate=2026-06-09
```

规则：

- 日期格式为 `yyyy-MM-dd`。
- `startDate` 和 `endDate` 都可选，不传时保持 v0.2 全量累计行为。
- `endDate` 按当天结束处理，后端转换为下一个自然日的开区间。
- `startDate > endDate` 时返回 400。

统计口径：

- 总览 PV/UV/UIP 按 `visit_event.created_at` 过滤。
- 结果生成数按 `user_result.created_at` 过滤。
- 短链生成数和短链列表按 `short_link.created_at` 过滤。
- 短链列表中的 PV/UV/UIP 按该短链的 `SHORT_LINK_VISIT` 事件过滤。
- 短链访问详情只展示筛选范围内的 `SHORT_LINK_VISIT`。

## 5. 测试覆盖

v0.3 新增或扩展：

- 后台日期筛选集成测试：当天有数据、未来日期为空、非法日期范围返回 400。
- external provider 请求体测试：确认 `domain`、`originUrl`、`gid` 等字段正确。
- external RestClient 测试：无端口捕获 HTTP 请求，验证路径、JSON body 和 `username/userId/realName` header。
- 原 v0.1/v0.2 主链路测试继续通过，默认 `internal` 模式不回退。
